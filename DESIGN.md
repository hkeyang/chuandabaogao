设计交互与开发规划 v1.0
1. 项目目标
本次改版目标不是做一个复杂 App，而是把现有 H5 轻应用升级成：

闲鱼购买兑换码 + H5 兑换权益 + 上传照片 + 生成单张高信息密度报告 + 结果页继续生成/分享

核心体验关键词：


轻量


好买


好懂


好生成


好保存


好分享


好复购


核心视觉关键词：


小红书种草感


女生用户友好


粉白奶油色


信息图报告感


分享卡片感


不像工具后台


不像严肃测评


像“我的专属变美报告”



2. 本期设计原则
2.1 页面不要太复杂
当前产品是 H5，不适合做复杂账号体系、复杂订单体系、复杂多页报告系统。
本期设计原则：

页面轻、路径短、状态清楚。

用户只需要理解三件事：


我买了什么券码


我还能生成什么报告


我生成完之后还能继续看什么



2.2 报告图是核心资产
页面 UI 是辅助，真正让用户愿意传播的是报告图。
因此设计优先级：
报告图质量 > 结果页体验 > 购买页转化 > 首页种草 > 其他细节
报告图必须做到：


一眼好看


主题准确


信息密度高


风格有记忆点


有小红书分享欲


每种专题视觉上能明显区分



2.3 价格差异靠“权益”和“内容范围”，不是靠功能复杂度
三个商品：
单次专题报告券 ¥1.9三次探索卡 ¥4.9全案探索卡 ¥9.9
设计上要让用户感受到：


¥1.9：试一个方向


¥4.9：多试三个方向


¥9.9：综合报告 + 三次专题，最完整


重点突出：

综合形象报告仅全案探索卡可生成。


3. 整体用户路径
3.1 主路径
首页→ 购买页 / 输入兑换码→ 闲鱼购买兑换码→ 回首页兑换→ 兑换成功页→ 选择报告类型→ 上传照片→ 选择生成偏好→ 确认生成→ 生成进度→ 查看结果→ 继续生成 / 下载 / 分享

3.2 三类用户路径
A. 新用户，先买 ¥1.9
首页看到报告预览→ 点击购买兑换码→ 进入购买页→ 买单次专题报告券→ 回首页兑换→ 选择 1 个专题→ 生成 1 张报告→ 结果页引导买三次探索卡 / 全案探索卡
设计重点：


首页要种草


购买页要让 ¥1.9 看起来门槛极低


结果页要引导“继续探索”



B. 想多试的用户，买 ¥4.9
兑换三次探索卡→ 剩余专题次数 3→ 选择专题→ 上传照片生成→ 查看结果→ 继续生成下一个专题→ 剩余专题次数 2 / 1 / 0
设计重点：


每次生成后展示剩余次数


推荐“下一个最适合生成的专题”


强调“同一张照片可继续探索不同方向”



C. 想完整方案的用户，买 ¥9.9
兑换全案探索卡→ 综合报告 1 + 专题报告 3→ 默认推荐先生成综合报告→ 查看综合报告→ 继续生成专题深挖
设计重点：


全案卡要显得更高级


兑换成功页要提示“建议先生成综合报告”


结果页要突出“综合报告已完成，继续生成专题”
